国内走国内 CDN，国外走 Cloudflare｜网站加速分流完整教程

Alima2025-06-102025-06-10

对于拥有全球用户的网站而言，如何确保不同地区访客都能获得快速、稳定的访问体验，是一个核心挑战。特别是，如何同时优化中国大陆和海外用户的访问速度，往往需要一套精巧的架构。

本教程将提供一个完整、可行的解决方案，通过 DNS 智能分流 与 多 CDN 协同 的方式，实现以下终极目标：

  • 中国大陆用户： 访问请求被智能引导至响应迅速的国内 CDN，享受极致的加载速度和稳定性。
  • 海外用户： 访问请求交由全球领先的 Cloudflare CDN 处理，利用其强大的国际链路质量和安全防护能力。
  • 统一源站： 所有 CDN 的请求最终都回源到同一台服务器，简化维护，确保内容一致。

整体架构与访问流程

在深入技术细节之前，让我们先理解整个系统的访问流程。当一个用户尝试访问我们的网站 www.haib.top 时，会发生以下情况：

这个架构的核心在于华为云 DNS 充当了“交通警察”的角色，根据访客的地理位置，将他们导向最合适的“高速公路”（国内 CDN 或 Cloudflare），而所有的“高速公路”最终都通向同一个“目的地”（你的源站服务器）。

准备工作

在开始配置前，请确保您已准备好以下资源：

项目 内容 说明
VPS 公网 IP 192.227.223.26 您的网站内容实际部署的服务器。
主域名 haib.top 建议在 Cloudflare 托管，方便管理。
访问子域名 www.haib.top 用户实际在浏览器中输入的地址。
回源域名 cdn.280222.xyz 一个专门用于回源的域名，也托管在 Cloudflare。
国内 CDN 平台 任意国内 CDN 厂商 用于为大陆用户加速。
DNS 分流服务商 华为云 DNS 具备按线路解析能力，是实现智能分流的关键。
网站协议 HTTPS 全站加密是现代网站的标配，也是本教程的基础。

详细实现步骤

第 1 步：配置源站服务器 (VPS)

源站是所有数据的起点，必须确保其稳定运行并配置正确。

  1. 部署网站： 在 IP 地址为 192.227.223.26 的 VPS 上部署好您的网站应用。
  2. 配置 Web 服务器： 配置 Nginx 或其他 Web 服务器，使其能够响应针对域名 cdn.280222.xyz 的请求。这是因为所有 CDN 回源时都会使用这个主机名。
  3. 启用 HTTPS： 为 cdn.280222.xyz 域名签发 SSL 证书（例如使用 Let’s Encrypt），并配置 Web 服务器强制启用 HTTPS。确保 http 请求能自动跳转到 https。

第 2 步：配置 Cloudflare (全球 CDN 与回源网关)

Cloudflare 在此架构中扮演双重角色：既是面向海外用户的 CDN，也是国内 CDN 的安全回源网关。

2.1 将回源域名指向 VPS

  1. 登录 Cloudflare 面板，选择 280222.xyz 域名。
  2. 进入 DNS -> Records 页面，添加一条 A 记录：
    • 类型 (Type): A
    • 名称 (Name): cdn
    • IPv4 地址 (Content): 192.227.223.26 (您的 VPS 公网 IP)
    • 代理状态 (Proxy status): **已代理 (Proxied)**，确保橙色云朵亮起。

2.2 配置 Cloudflare 自定义主机名

这一步是让 Cloudflare 能够处理对 cdn.280222.xyz 的 HTTPS 请求。

  1. 在 280222.xyz 域名的面板中，找到 SSL/TLS -> Custom Hostnames。
  2. 设置回退源 (Fallback Origin): 输入 cdn.280222.xyz。这是当没有匹配的自定义主机名时，请求将被路由到的地方。
  3. 点击 Add Custom Hostname，并填写信息：
    • Custom Hostname: cdn.280222.xyz
    • SSL/TLS certificate type: 选择 Cloudflare 提供的免费证书。
  4. 按照 Cloudflare 的提示，它会提供一个 TXT 验证记录。回到 DNS -> Records 页面，将这条 TXT 记录添加进去。
  5. 等待几分钟，直到自定义主机名的状态变为 Active，这表示 Cloudflare 已成功为其颁发 SSL 证书。
  6. 最后，在 SSL/TLS -> Overview 页面，将 SSL 加密模式设置为 **完全 (严格) (Full (Strict))**，确保从用户到 Cloudflare 再到源站的全链路加密。同时，建议开启“始终使用 HTTPS”和“自动 HTTPS 重写”。

第 3 步：设置 DNS 解析权，将 www 子域交给华为云

现在，我们需要将用户访问的入口 www.haib.top 的 DNS 解析权从 Cloudflare 移交给支持分流的华为云 DNS。

  1. 登录 Cloudflare，进入 haib.top 域名的 DNS 管理页面。
  2. 重要： 如果存在 www 的 A 记录或 CNAME 记录，请务必 删除 它。
  3. 添加 NS 记录，将 www 这个子域名的解析权限指向华为云 DNS。你需要添加 4 条 NS 记录：
    • 类型: NS, 名称: www, 内容: ns1.huaweicloud-dns.com
    • 类型: NS, 名称: www, 内容: ns1.huaweicloud-dns.cn
    • 类型: NS, 名称: www, 内容: ns1.huaweicloud-dns.net
    • 类型: NS, 名称: www, 内容: ns1.huaweicloud-dns.org

第 4 步：配置国内 CDN 平台

登录您选择的国内 CDN 厂商控制台，为 www.haib.top 添加加速配置。这是最关键的配置环节之一。

字段 填写内容 解释
加速域名 www.haib.top 用户访问的域名。
域名协议 HTTPS 申请或上传 www.haib.top 的 SSL 证书。
源站协议 HTTPS 回源请求使用 HTTPS 协议。
源站地址 cdn.280222.xyz:443 核心配置！ 让国内 CDN 去请求 Cloudflare，而不是直接请求 IP。
回源主机 cdn.280222.xyz 核心配置！ CDN 回源时使用的 Host 头，必须与 Cloudflare 中配置的域名一致。
缓存设置 按需配置 根据你的网站内容（静态/动态）设置合适的缓存规则。

配置完成后，CDN 平台会提供一个 CNAME 地址，例如 abc.cdnprovider.com。记下这个地址。

第 5 步：在华为云 DNS 配置多线路分流

最后一步，我们来扮演“交通警察”，设置智能分流规则。

  1. 登录华为云 DNS 控制台，找到并进入 www.haib.top 的解析管理页面。
  2. 添加以下 三条 CNAME 记录，它们的记录名称（主机记录）都是 www，但线路类型不同：
线路类型 类型 记录值 说明
境内 CNAME abc.cdnprovider.com 国内 CDN 的 CNAME 地址，覆盖中国大陆用户。
境外 CNAME cdn.280222.xyz Cloudflare 回源域名，覆盖所有海外用户。
默认 CNAME abc.cdnprovider.com 兜底线路，建议设为国内 CDN，确保解析稳定。

保存后，等待几分钟让 DNS 记录生效。

验证与总结

至此，整个分流加速架构已部署完毕。你可以使用 itdog.cn 等多地 Ping 工具进行测试：

  • 从中国大陆的节点测试 www.haib.top，应该会解析到国内 CDN 的 CNAME。
  • 从海外节点测试，则会解析到 cdn.280222.xyz。

通过这套精密的 DNS 与 CDN 组合拳，我们成功地为网站构建了一套高性能的全球访问网络，不仅极大提升了国内外用户的访问体验，还通过 Cloudflare 增强了网站的安全性和可用性，同时保持了源站维护的简洁性。
